# 0.11.0 Bot Safety Summary

## What shipped

Community Engine `0.11.0` now includes a built-in, local-first bot-safety baseline.

This release adds:

- signed submission proofs for protected forms
- hidden honeypot fields
- timing and replay checks
- challenge issuance throttling
- explicit first-party robot access with scoped tokens

## Why it matters

Before this work, host apps could add stronger protection, but the default engine path did not provide a complete baseline on its own.

Now a fresh CE deployment starts from a safer position for:

- registration
- membership requests
- safety reports

## What did **not** change

- Community Engine is not claiming to solve all bot abuse by itself.
- Organizers still review membership requests and safety reports.
- Turnstile remains optional for host apps that need stronger external human verification.
