# 0.11.0 Setup Wizard and Platform Bootstrap Evidence

`0.11.0` hardens the host-platform bootstrap path so the first-run wizard behaves like a one-time platform initialization surface instead of a reusable public entry point.

## What shipped

- the setup wizard resolves to the host-platform bootstrap flow
- the platform-details step reuses the existing host platform instead of creating duplicates
- wizard completion blocks re-entry for both signed-in and anonymous requests
- the host platform defaults to a protected, invitation-oriented posture during bootstrap

## Key implementation points

- [SetupWizardController](../../app/controllers/better_together/setup_wizard_controller.rb) redirects the host setup wizard to the platform-details step and blocks entry when the wizard is already complete.
- [SetupWizardStepsController](../../app/controllers/better_together/setup_wizard_steps_controller.rb) uses `find_or_initialize_by(host: true)` in `base_platform`, which is the core duplicate-prevention behavior for host-platform setup.
- The same controller applies `ensure_setup_wizard_incomplete` to all setup actions, including the POST paths that create the host platform and first admin, so the flow cannot be replayed after completion.
- The [platform_details template](../../app/views/better_together/wizard_step_definitions/host_setup/platform_details.html.erb) is the main reviewer-facing UI for the bootstrap step.
