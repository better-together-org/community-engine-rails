# 0.11.0 Asset Capture Plan

This plan turns the remaining `0.11.0` release-packet gaps into concrete follow-up work. It only covers branch-tip assets that are still missing after the release inventory pass.

## Working rules

- Prefer existing screenshot specs when the shipped UI is already covered by a stable review flow.
- Create new release-focused screenshot specs only when the current review specs do not expose the correct page or state.
- Do not invent screenshots for UI that does not exist on the release branch tip.
- For non-UI gaps, add docs or diagrams instead of forcing synthetic capture work.

- source UI:

  - sort selector
  - storage usage summary
  - card grid with `Copy URL` and `Insert`

Recommended implementation:

- add a new docs screenshot spec:

### Block image-library picker

This needs a dedicated release-focused spec. The current `pr_1490` spec covers hero and checklist editor behavior, but not the upload-library insertion flow.

- likely source route:
  - block editor new-form flow for an image-capable block type
- supporting shipped UI:
  - uploads gallery `Insert` button carries the blob `signed_id`
- target slugs:
  - `release_0_11_0_block_image_library`

Recommended implementation:

- add a new docs screenshot spec:
  - `spec/docs_screenshots/better_together/release_0_11_0_block_image_library_spec.rb`
- seed:
  - one or more image uploads

- capture the state where the library is visible and clearly shows the selection/insertion affordance
