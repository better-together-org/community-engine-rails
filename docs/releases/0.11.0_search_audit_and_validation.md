# 0.11.0 Search Audit and Validation

`0.11.0` expands the search lane beyond basic query execution. The release now includes audit capture, backend visibility, and an explicit registry-backed default search path that is easier to review and safer to operate.

## What shipped

- search queries are captured after server-side execution, subject to platform privacy controls
- captured queries can be disabled entirely or stored in hashed form
- the search backend is registry-based and defaults to `pg_search` on this branch, with a database-backed fallback for models that have not yet been upgraded to a `pg_search` scope
- release validation now needs to confirm the default backend story and audit behavior instead of implying that Elasticsearch is the primary `0.11.0` path

## Key implementation points

- [SearchQueryCaptureService](../../app/services/better_together/metrics/search_query_capture_service.rb) normalizes queries, respects platform-level analytics settings, and can hash retained search text with a `sha256:` prefix.
- [TrackSearchQueryJob](../../app/jobs/better_together/metrics/track_search_query_job.rb) and the metrics system docs tie search-query retention into the analytics/privacy lane.
- [Search](../../app/services/better_together/search.rb) makes the backend selection explicit and defaults the branch to `pg_search`.
- [PgSearchBackend](../../app/services/better_together/search/pg_search_backend.rb) documents the shipped backend behavior, including the database-backed fallback path used when a model has not yet been upgraded to a `pg_search` scope.
- [Registry](../../app/services/better_together/search/registry.rb) defines the searchable model surface used by the global search lane.
- The runtime default is also visible in [.env.dev](../../.env.dev), which now sets `SEARCH_BACKEND='pg_search'` and leaves the old Elasticsearch lane as optional commentary rather than the primary release story.
- The real-backend validation path is covered by [search_matching_spec.rb](../../spec/integration/better_together/search_matching_spec.rb), [pg_search_backend_spec.rb](../../spec/services/better_together/search/pg_search_backend_spec.rb), and the search-audit work called out in the compare range.
