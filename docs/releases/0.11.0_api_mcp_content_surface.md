# 0.11.0 API and MCP Content Surface

`0.11.0` materially expands the machine-facing content surface. The release now includes JSON:API endpoints for blocks/page-blocks and a broader MCP tool layer for searching, updating, and publishing content while still respecting policy scope and platform privacy.

## What shipped

- JSON:API endpoints for content blocks and page blocks
- MCP tools for searching pages and posts
- MCP tools for updating and publishing posts
- shared content/page-block groundwork that supports the new block families and later rollout work

## Key implementation points

- [BlocksController API](../../app/controllers/better_together/api/v1/blocks_controller.rb) and [PageBlocksController API](../../app/controllers/better_together/api/v1/page_blocks_controller.rb) expose the structured block/page-block lane over JSON:API.
- The MCP surface includes:
  - [SearchPagesTool](../../app/tools/better_together/mcp/search_pages_tool.rb)
  - [SearchPostsTool](../../app/tools/better_together/mcp/search_posts_tool.rb)
  - [UpdatePostTool](../../app/tools/better_together/mcp/update_post_tool.rb)
  - [PublishPostTool](../../app/tools/better_together/mcp/publish_post_tool.rb)
- Authorization and privacy filtering remain policy-scoped through the MCP integration path described in [mcp_integration.md](../development/mcp_integration.md).
