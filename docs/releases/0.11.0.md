## What changed since 0.10.0

### 1. Multi-tenant platform architecture and federation

`0.11.0` expands Community Engine from a single-host application model into a platform-aware engine with tenant context, platform memberships, platform connections, and mirrored-content flows.

Key release surfaces:

- host-platform and tenant-aware request context
- federation connections and OAuth-backed trust flows
- mirrored pages, posts, and events with identifier disambiguation
- authorship opt-in for federated bylines

Related docs:

- [Multi-Tenancy Upgrade Guide](../upgrade/multi-tenancy-upgrade.md)
- [Federated seed and sync handoff](../implementation/multi_tenancy/federated_seed_and_sync_handoff_2026-03-12.md)
- [Federated RBAC reassessment and coverage plan](../implementation/multi_tenancy/federated_rbac_reassessment_and_coverage_plan.md)

### 2. CMS blocks, JSON:API, and MCP management

The content system now has a much broader block model, API surface, and MCP tooling. This release also tightens the block-picker/accessibility pass and adds safer embedded-content rendering.

Key release surfaces:

- expanded block catalog and page-block management flows
- JSON:API coverage for blocks and page blocks
- MCP tooling for block management
- accessibility fixes for block-picker labels
- safe iframe block support and shared embed rendering path

Related docs:

- [Content block completeness checklist](../content-block-completeness-checklist.md)
- [MCP integration guide](../api/mcp-integration-guide.md)

### 3. Privacy, safety, and personal-data stewardship

`0.11.0` continues the privacy-hardening direction by improving data export flows, federation authorship consent, invitation/policy handling, and safety reporting.

Key release surfaces:

- personal seed exports and scoped access policies
- accessible safety reporting workflow with linked safety cases, reviewer lanes, and closure summaries
- stronger policy and scope behavior for invitation and membership contexts
- merged `ApplicationPolicy::Scope` fix so scope subclasses can safely consume keyword context

Related docs:

### 4. End-to-end encrypted conversations remain beta-gated

`0.11.0` includes the Signal-based E2EE conversation stack, but the current rollout posture is explicit opt-in rather than general availability. Current code reads `BETTER_TOGETHER_E2EE_MESSAGING_ENABLED` before enabling the E2EE messaging path, and the session bootstrap is rendered from conversation UI rather than mounted globally from the main application layout.

Key release surfaces:

- disabled-by-default activation via `BETTER_TOGETHER_E2EE_MESSAGING_ENABLED`
- conversation-scoped bootstrap and send-form integration instead of site-wide E2EE mounting
- plaintext fallback for conversations that are not E2EE-ready or are missing participant key material
- rollout guidance limited to opted-in deployments and intended conversation surfaces
- security model follow-ups still tracked for V9 (distribution ID randomness) and V10 (HKDF salt handling) before any broader rollout posture changes

The release rounds out the main browsing surfaces with search and discovery work across posts, pages, and events, plus Elasticsearch 8 validation and reindex support. The changelog remains the most complete ledger for the newer posts/events filter services; the related docs below cover the adjacent browsing and search surfaces already documented on `main`.

Key release surfaces:

- global search for pages and posts, with suggestions and search-query metrics
- event discovery through draft/upcoming/ongoing/past timing scopes
- Elasticsearch 8 validation, audit, and reindex support for the current registry
- incremental metrics coverage improvements

## Late release-candidate fixes merged during stabilization

These fixes landed during release-candidate stabilization and are included in the current `0.11.0` candidate.

### `#1421` — safe iframe embeds and CSP-managed origin lists

This slice adds:

- a dedicated `IframeBlock`
- host-platform `frame-src`, `frame-ancestors`, and `img-src` origin settings
- blocked-state fallback UX instead of silent iframe failure
- host-aware fragment cache keys for iframe/video embeds

Documentation companion:

- [Embedded content and CSP controls](../platform_organizers/embedded_content_and_csp.md)

### `#1456` — preserve keyword scope options for policy subclasses

This slice is intentionally small and internal-facing:

- `ApplicationPolicy::Scope` now keeps arbitrary keyword options
- scope subclasses can consume context-style inputs without losing `invitation_token`
- regression coverage documents the intended behavior

This affects release correctness more than end-user behavior, so it is covered in the changelog rather than a standalone operator guide.
