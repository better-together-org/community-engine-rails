# User Support Procedures

## Intake
- Track support requests (email, form, or in-app) with timestamps and requester info.
- Collect key details: URL, steps to reproduce, expected vs actual, screenshots/logs.

## Resolution Steps
- Verify platform status and recent deployments.
- Reproduce issues in a safe environment when possible.
- Escalate authentication/authorization problems to platform managers.
- Communicate progress and resolution clearly with the user.

## Aftercare
- Document fixes or workarounds in internal notes.
- Add FAQs or help content when patterns emerge.
- Close tickets only after user confirmation where practical.
