# Safety and Reporting Tools

**Target Audience:** All community members  
**Document Type:** User Guide  
**Last Updated:** April 8, 2026

## Overview

This guide is the main starting point for the platform's current safety tools.

Use these tools when you need to:

- create distance from another user
- report harm, abuse, or boundary violations
- understand what happens after you submit a report
- decide what to do in urgent or emergency situations

> **Important:** The platform can support follow-up, documentation, and moderation review. It is **not** an emergency response service.

## Start Here

### If you need space from someone right now

Read [Blocking and Boundaries](blocking_and_boundaries.md).

Use blocking when you want to reduce contact or make it harder for a specific person to interact with you on the platform.

### If you need to tell the platform about harm or a safety concern

Read [Reporting Harm and Safety Concerns](reporting_harm_and_safety_concerns.md).

Use reporting when you want the platform or safety team to review behavior, content, or patterns of harm.

### If you already submitted a report

Read [After You Report](after_you_report.md).

This explains what you can currently see in the report history and report detail pages, and how to add follow-up evidence or appeal context from the report page.

### If you need help understanding privacy or consent choices

Read [Privacy and Safety Preferences](privacy_and_safety_preferences.md).

This explains the current privacy-related safety controls and the consent fields in the reporting flow.

### If the situation is urgent or someone may be in immediate danger

Read [Emergency and Urgent Situations](emergency_and_urgent_situations.md).

## Current Safety Features

The current end-user safety feature set includes:

- personal blocking tools
- structured safety reporting
- shared content-surface reporting entry points
- section-level reporting from supported content blocks
- report history and report detail pages
- participant-visible follow-up notes and appeal/evidence entry
- safety-related consent options in the report form
- privacy controls in some user-facing areas

The platform is still growing. Some older documentation describes moderation workflows in more detail than the current user-facing interface shows. The pages in this section are written to match the current implementation as closely as possible.

## Visual Guides

The main guides in this section include screenshots of the current interface where they help explain a task. Every visual walkthrough is paired with plain-language text so no user has to rely on images or motion alone.
